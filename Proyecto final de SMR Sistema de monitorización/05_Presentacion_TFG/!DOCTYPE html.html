Sistema de Monitorización y Control de Red Doméstica

Proyecto Final – 2º SMR

Ricardo Gordo

Objetivo del Proyecto

  • Simular una red doméstica completa.
  • Monitorizar equipos y servicios en tiempo real.
  • Detectar fallos, caídas y problemas de rendimiento.
  • Automatizar alertas y notificaciones.
  • Aprender herramientas profesionales de administración.

Arquitectura del Sistema

  • VirtualBox: máquinas virtuales.
  • GNS3: topología de red.
  • Servidor Zabbix + MariaDB.
  • Clientes Windows y Ubuntu.
  • ServidorServicios con DHCP, DNS y Web.

Instalación y Configuración

  • Instalación de Ubuntu Server.
  • Instalación de Zabbix Server + MariaDB.
  • Instalación de agentes en cada equipo.
  • Asociación de plantillas de monitorización.

Monitorización

  • CPU, RAM, Disco.
  • Ping y latencia.
  • Interfaces de red.
  • Servicios del ServidorServicios.
  • Gráficos y métricas en tiempo real.

Triggers Personalizados

  • CPU alta.
  • RAM baja.
  • Disco bajo.
  • Ping caído.
  • Pérdida de paquetes.

Pruebas de Estrés

  • CPU: stress --cpu 4
  • RAM: stress --vm 1 --vm-bytes 6G
  • Disco: dd if=/dev/zero of=/tmp/testfile
  • Validación manual con top, free -h, df -h.

Alertas y Acciones Automáticas

  • Panel de problemas en Zabbix.
  • Acción “Problema” configurada.
  • Envío de correo a administradores.
  • Notificaciones en tiempo real.

Resultado Final

  • Red doméstica completamente simulada.
  • Monitorización profesional con Zabbix.
  • Detección de fallos en tiempo real.
  • Automatización de alertas.
  • Documentación completa del sistema.
